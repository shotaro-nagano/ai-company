# PixelYen セットアップ手順(人間の作業・1手順ずつ)

建国日にAIと一緒に進めるチェックリストです。**上から順に、1つずつ**やります。
分からない用語が出たら、各手順の(用語)説明を読んでください。

---

## Step 1: GitHubリポジトリの作成(5分)

(用語)リポジトリ = プロジェクトのファイル置き場。公開リポジトリは誰でも中身を見られます。

1. https://github.com/new を開く
2. 「Repository name」に `ai-company` と入力
3. 「Public」を選ぶ(公開経営=憲法第10条。公開だとActionsも無料)
4. ほかのチェックは**すべて空のまま**、緑の「Create repository」ボタンを押す
5. 表示されたページのURL(`https://github.com/あなたのID/ai-company`)をAIに伝える

## Step 2: 2要素認証の有効化(5分・未設定の場合)

(用語)2要素認証(2FA)= パスワードに加えスマホでの確認を必須にする仕組み。乗っ取り対策。

1. https://github.com/settings/security を開く
2. 「Two-factor authentication」の「Enable two-factor authentication」を押す
3. スマホの認証アプリ(Google Authenticator等)でQRコードを読み取り、6桁コードを入力
4. **リカバリーコードを必ず保存**(スクリーンショット+印刷推奨)

## Step 3: Claude CodeのCI用トークン発行(3分)

(用語)トークン = AIがあなたの代わりにClaudeへログインするための合言葉。**1年で失効**します(失効前にマモルが#承認待ちで知らせます)。

1. ターミナル(このClaude Codeを動かしている画面)で、AIが `claude setup-token` を実行する
2. ブラウザが開くので「許可」を押す
3. ターミナルに表示された `sk-ant-…` で始まる長い文字列をコピー(**誰にも見せない・どこにも貼らない**。次のStepでだけ使う)

## Step 4: GitHub Secrets への登録(トークンごとに2分)

(用語)Secrets = 合言葉の金庫。ここに入れた値はワークフローだけが読め、公開されません。

1. `https://github.com/あなたのID/ai-company/settings/secrets/actions` を開く
2. 緑の「New repository secret」ボタンを押す
3. 「Name」に `CLAUDE_CODE_OAUTH_TOKEN`、「Secret」にStep 3の文字列を貼り付け、「Add secret」

以降、取得できたものから同じ手順で追加(未取得はスキップ可・後回しでOK):

| Name | 値の入手先 |
|---|---|
| DISCORD_WEBHOOK_APPROVAL 〜 _ERROR(6本) | Step 6 |
| DISCORD_BOT_TOKEN | Step 7 |
| STRIPE_API_KEY | Stripe審査通過後(ダッシュボード→開発者→APIキー→シークレットキー) |
| ITCH_API_KEY | itch.io→Settings→API keys |
| GMAIL_CLIENT_ID / GMAIL_CLIENT_SECRET / GMAIL_OAUTH_REFRESH_TOKEN | サポートGmail開設後(AIが手順を案内) |

## Step 5: Discordサーバーのチャンネル作成(10分)

サーバー「PixelYen」に、テキストチャンネルを8本作ります。

1. Discordの左のサーバー名の下、「テキストチャンネル」の右の「+」を押す
2. チャンネル名を入れて「チャンネルを作成」— 以下を繰り返し:
   `承認待ち` / `売上通知` / `週次レポート` / `経営会議` / `作業ログ` / `エラー` / `社長室` / `general`(最初からあれば流用)

## Step 6: Webhookの作成(チャンネルごとに2分・6本)

(用語)Webhook = 「このURLに文章を送るとチャンネルに投稿される」仕組み。社員の報告はこれで届きます。

対象: 承認待ち / 売上通知 / 週次レポート / 経営会議 / 作業ログ / エラー の6チャンネル

1. チャンネル名の右の⚙(設定)を押す
2. 左メニュー「連携サービス」→「ウェブフックを作成」
3. 名前は「PixelYen」など適当でOK →「ウェブフックURLをコピー」
4. Step 4の手順で Secrets に登録(Name は下表のとおり):

| チャンネル | Secret名 |
|---|---|
| 承認待ち | DISCORD_WEBHOOK_APPROVAL |
| 売上通知 | DISCORD_WEBHOOK_SALES |
| 週次レポート | DISCORD_WEBHOOK_WEEKLY |
| 経営会議 | DISCORD_WEBHOOK_MEETING |
| 作業ログ | DISCORD_WEBHOOK_LOG |
| エラー | DISCORD_WEBHOOK_ERROR |

## Step 7: Discord Botの作成(10分)

(用語)Bot = 読み取り係。#承認待ちのあなたの✅と、#社長室のあなたの指示を、セッション開始時に読みます(常駐しません)。

1. https://discord.com/developers/applications を開き「New Application」→ 名前「PixelYen Reader」→ Create
2. 左メニュー「Bot」→「Reset Token」→ 表示されたトークンをコピー → Secrets に `DISCORD_BOT_TOKEN` で登録
3. 同じ画面の「Privileged Gateway Intents」で「MESSAGE CONTENT INTENT」をオンにして Save
4. 左メニュー「OAuth2」→「OAuth2 URL Generator」で:
   - SCOPES: `bot` にチェック
   - BOT PERMISSIONS: `View Channels` `Read Message History` にチェック
   - 下に出たURLを開き、サーバー「PixelYen」を選んで「認証」
5. **チャンネルIDの取得**: Discordの設定→詳細設定→「開発者モード」をオン。
   `承認待ち` と `社長室` のチャンネル名を右クリック→「チャンネルIDをコピー」して、AIに2つのIDを伝える
   (AIが `state/discord_channels.json` に記録します。IDは秘密情報ではありません)

## Step 8: GitHub Pages の有効化(2分)

(用語)GitHub Pages = リポジトリから無料でWebサイトを公開する仕組み。販売サイトとダッシュボードが載ります。

1. `https://github.com/あなたのID/ai-company/settings/pages` を開く
2. 「Build and deployment」の「Source」で **「GitHub Actions」** を選ぶ(保存ボタンは不要・選ぶだけ)

## Step 9: Actions の権限確認(2分)

1. `https://github.com/あなたのID/ai-company/settings/actions` を開く
2. 「Workflow permissions」で **「Read and write permissions」** を選択し「Save」
   (社員が state/ の更新をコミットするのに必要です)

---

これで建国のオンライン設定は完了。あとはAIが試運転(Week 1チェックリスト)を実行します。
