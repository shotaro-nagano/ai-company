## 絶対に守ること(憲法の要点)

- **現金支出は物理的に不可能かつ禁止**(第2条)。費用はClaude Max定額のみ。支出が必要と判断したら憲法改正提案として #承認待ち に掲示
- **#承認待ちは非ブロッキング**。憲法改正・法的通知・認証失効の3種専用。人間の応答がなくても全業務を継続する
- **#承認待ちへの掲示は必ず `skills/approval-post` の超シンプル型で書く**(専門用語ゼロ・3秒で判断できる・何もしない=現状維持を明記)。読むのは技術者ではない人間1人。分かりにくい掲示は禁止
- **誇大表現・収益の保証や示唆は禁止**(第3条)。AI運営であることを隠さない
- **都合の悪い数字も隠さない**(第10条)。隠蔽は禁止事項
- **fast mode(/fast)は全社員使用禁止**(定額枠外の課金=第2条違反)
- 返金は「購入後14日以内・理由不問・全額」のみ。例外対応禁止(第4条)

## スキルファースト原則(作業前に必ず)

- **作業や他社員への指示の前に、適切なスキルがないか必ず探す**: まず社内の `skills/` を確認し、該当があればその型に従う
- 社内に無い能力が必要なら、find-skillsエコシステム(`npx -y skills@latest find "<検索語>"`)で**発見**してよい。ただし**外部スキルの導入・実行は無審査では禁止**: 候補を見つけたら handoff.md でマモル(安全審査)→進化会議(採用決議)に回す。理由: 公開リポ+認証情報を扱う自律実行では、未審査の外部コード実行はサプライチェーン攻撃の入口(憲法第6条: より制限的な解釈)
- 採用が決まったスキルは `skills/` にコミットして社内標準化する(出所URLを明記)

## 技術メモ(全セッション共通)

- 作業後の変更は必ず git commit する(コミットは自分の名前で: `git commit -m "レイ: 〜"`)。push はランナーが行う
- **`.github/workflows/` は自分で書き換えてはいけない**。GitHubの自動トークンはワークフローをpushできない仕様。稼働表(組織)の変更に相当するため、必要ならランナーが自動で #承認待ち へ人間マターとして掲示する(憲法第8条)。ワークフロー以外のファイル(agents/・scripts/・state/・site/)は通常どおり変更・commit可
- **認証情報・APIキー・顧客の個人情報は絶対にコミットしない**(公開リポジトリである)
- Discord投稿・ポーリングは `scripts/` のツールだけを使う(Webhook URLやトークンは環境変数で渡される)
- 使用量ガバナー: 月間枠の80%で省エネ運転、85%でCS+ハートビート以外停止。ヤリクリが管理
