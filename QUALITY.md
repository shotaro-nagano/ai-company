# QUALITY.md — 運用品質基準書

## 大原則: 品質はモデルではなく文脈が決める

どれほど賢いモデルも、文脈が欠けたセッションでは平凡な仕事しかできない。本書は、すべてのセッションに「会社の全体像」を持たせ、単発の作業を積み重ねても品質が劣化しない仕組みを定める。

## 1. CLAUDE.md — 全セッションの入口

- リポジトリ直下に `CLAUDE.md` を置く(Claude Codeが自動で読み込む)
- 記載内容: 憲法・組織・VISION・本書の要約と参照先 / 現在のPhaseとレベル / 直近の重要決定3件 / 今月の最優先目標
- **シオリが週1で最新化する。** 古いCLAUDE.mdは全社員の判断を誤らせる

## 2. ロールプロンプトの基準

各社員のプロンプトは以下の6要素を必須とする。**「あなたはCEOです」だけのプロンプトは禁止。**

1. 役割と権限(何を決めてよいか)
2. 判断基準(迷ったときの優先順位。例: レイ「利益 > 速度 > 見栄え」)
3. 出力フォーマット(報告・成果物の型)
4. 良い例・悪い例(few-shot。各1つ以上)
5. 禁止事項(憲法の該当条文を参照)
6. 完了条件(何をもって仕事を終えたとするか)

## 3. セッションの標準手順(全社員共通)

1. `CLAUDE.md` → 自分のロールプロンプト → `state/` の順に読む
2. 前回セッションからの申し送りを確認する
3. 作業する
4. `state/` を更新し、次回への申し送りを書く
5. Discordへ所定フォーマットで報告する

## 4. モデル階級制(ヤリクリ管轄)

| 場面 / 社員 | モデル | effort |
|---|---|---|
| 進化会議・メツケ監査(月1) | Opus 4.8 | xhigh |
| 週次経営会議・完全ガイド等の重要文書執筆 | Opus 4.8 | high |
| レイの日次セッション | Opus 4.8 | medium |
| メキキ・オキテ・ヨミ・シラベ・ヒラメキ・カタリベ・ツクル | Sonnet 4.6 | high |
| ヒカリ・イロハ・ワタル・ツヅリ・ムスビ・カケツ・シオリ・マモル・カゾエ・ヤリクリ・マメ | Sonnet 4.6 | medium |
| マトメ(要約・ハートビート)・ヒロメ(日次投稿) | Haiku | low〜medium |

**運用ルール**

- effortは各社員のロールプロンプト(subagent frontmatter)に記載し、その社員の稼働時だけ自動適用する
- 例外的に深く考えるべき単発の局面では、プロンプト内に「ultrathink」を用いてよい
- **fast mode(/fast)は全社員使用禁止** — サブスクの定額枠外(extra usage)で課金されるため、憲法第2条(定額以外の支出禁止)違反となる
- 割当の変更権限はヤリクリが持つ: 月間枠の80%超過時、本表を1段階ずつ格下げ(Opus→Sonnet、high→medium)できる。格上げは進化会議の決議事項
- 原則: **判断の重さと失敗コストにモデルとeffortを合わせる。** 単純タスクへの高effortは過剰設計を招き、重要判断への低effortは品質を毀損する

## 5. 品質ゲート

- **出荷前レビュー(メキキ・必須):** チェックリスト運用 — 誤字脱字 / 動作確認 / 誇大・保証表現なし(憲法第3条)/ 価格表記 / 特商法ページへの導線。指摘には必ず「直し方」を添える
- **相互レビュー:** 重要成果物(商品・週次レポート・対外文書)は制作者以外の1名が確認する

## 6. Skills — 社内ノウハウの結晶化

- 繰り返す業務は `skills/` 配下に SKILL.md 化する(例: 週次レポートの書き方、KDP入稿手順、販売ページの型、レビュー返信の型)
- うまくいったやり方は進化会議で「Skill昇格」を審議し、以後の全セッションで再利用する
- Skillは「一度うまくいった品質」を会社の標準に固定する装置である

## 7. 品質の計測

- 人間の 👍/👎 リアクション、返金率、購入者レビュー評価を品質KPIとして `state/kpi.md` に記録する
- 進化会議(憲法第9条)は感想ではなくこのデータを根拠に改善する

## 8. 劣化の検知と防止

- **シオリ(週1):** `state/` の腐敗(古い情報・矛盾・肥大化)を掃除する。記憶の質=判断の質
- **メツケ(月1):** 憲法からの逸脱に加え、本書の基準からの逸脱(手抜きプロンプト、レビュー省略、申し送り欠落)を監査し人間に直接報告する
- 本書自体も進化会議の改善対象とする(改訂は人間の承認を要する)
