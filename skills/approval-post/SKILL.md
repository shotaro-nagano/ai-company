# #承認待ち の書き方(人間ファースト)

`#承認待ち` を見るのは技術者ではない人間1人だけ。**専門用語ゼロ・3秒で判断できる**こと。
長い経緯・条文番号・英語は書かない(必要なら「詳しくは #作業ログ」と一言添える)。

## 必ずこの型で書く

```
🟡 ご確認のお願い(<種別>)

【内容】<何をしたいか。中学生にもわかる1文>
【お願い】このメッセージに…
   ✅ を付ける → 賛成
   何もしない → 現状のまま(急がなくてOK)

<※必要なら注意書きを1行だけ>
```

- `<種別>` は「憲法の変更 / 認証の更新 / 法的なお知らせ」の3つのどれか(それ以外は #承認待ち に載せない)
- 「❌」を使ってもよいが、**「何もしない=現状維持」**を必ず明記する(放置が正しい選択肢だと伝える。人間を急かさない=非ブロッキング)
- お金・数字が絡むときは「AIはお金を使えないので、承認されても実際には使えません」など**不安を消す一言**を添える

## 良い例

```
🟡 ご確認のお願い(認証の更新)

【内容】DiscordのBotの合言葉が90日後に切れます。新しいものへの入れ替えをお願いしたいです。
【お願い】このメッセージに…
   ✅ を付ける → 「対応する」の合図(手順は私が案内します)
   何もしない → 期限が近づいたらまたお知らせします

※今すぐでなくて大丈夫です。切れる前ならいつでもOK。
```

## 悪い例(禁止)

```
第2条予算制約の緩和提案。DISCORD_WEBHOOK_URL失効に伴いTAT再発行が必要。
CI/CD headlessのOAuth precedenceを考慮し… ← ✗ 専門用語だらけ・何をすればいいか不明
```
(×: 人間が読めない。何を求められているか・何を押せばいいかが一目でわからない)

## 掲示後

- 掲示したことを #作業ログ にも一言残す
- マトメは #承認待ち に溜まった項目を、この型に沿って3行以内へ整理し直してよい(古い分かりにくい掲示の要約・置き換え)
