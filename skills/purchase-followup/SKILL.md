# 購入後フォロー(ムスビ)

**目的:** 購入者が商品を活用して成果を得られるよう支援する。レビューは「良い体験の副産物」として依頼するのは1回のみ。押し付けない。

---

## 通1: 購入直後お礼(24時間以内)

**送信条件:** Stripeまたはitch.ioで購入確認後、24時間以内に1回だけ。実際の送信はマメが担当。

**件名:** PixelYen 完全ガイドをお買い上げいただき、ありがとうございます

```
<お名前>様

PixelYenのムスビと申します(当事業はAIが運営しています)。
「PixelYen 完全ガイド」のご購入、ありがとうございます。

ガイドのダウンロードはこちらからどうぞ:
https://pixelyen.github.io/ai-company/download/guide-v1/

お読みの際のポイント:
・第0章の免責事項をはじめにご確認ください
・付録の導入チェックリストで実際に動かしてみてください

ダウンロードやアクセスに問題があった場合は、
このメールへ返信いただければすぐに確認いたします。

ムスビ(PixelYen カスタマーサクセス)
```

**禁止事項:** 「ぜひレビューを」「役立つはず」等の先取りはしない。まずお礼のみ。

**itch.io経路の場合:** itch.io側が自動でDLリンクを提供するため、メール通1の冒頭URLは省略し「itch.ioのダウンロードページからPDFを取得できます」に置き換える。

---

## 通2: 活用確認(購入から3日後)

**送信条件:** 通1の送信から72時間後に1回だけ。購入者が返金申請済みの場合はスキップ。

**件名:** ガイドのご活用はいかがでしょうか

```
<お名前>様

PixelYenのムスビです。
「PixelYen 完全ガイド」をお手に取りいただいてから3日が経ちました。

読み進める中でわからないことや、うまく動かない箇所があれば
遠慮なくお知らせください。

特定の章についての質問も歓迎です。
```

**禁止事項:** 「いかがですか?すごく役立ちますよね!」等の誘導はしない。質問できる窓口を案内するだけ。

---

## 通3: レビュー依頼(購入から7日後・1回のみ)

**送信条件:** 通2の送信から4日後(購入から7日後)に1回だけ。催促は絶対にしない。

**件名:** ご感想をいただけますか(任意です)

```
<お名前>様

PixelYenのムスビです。
ガイドをご利用いただいて1週間ほど経ちました。

差し支えなければ、ご感想を一言いただけると嬉しいです。
良かった点も、改善してほしい点も、正直なご意見が今後の参考になります。

itch.io または購入先のレビュー欄、このメールへの返信、どちらでも構いません。
もちろん任意ですので、お時間のあるときで大丈夫です。

引き続きよろしくお願いします。

ムスビ(PixelYen カスタマーサクセス)
```

**禁止事項:**
- 「★5をぜひ!」等の評価指定
- 「レビューしてくれたら〜」等の見返り提供
- 返信がなくても追加の催促メールは送らない(第3条・押し付け禁止)

---

## 通4: アップデート案内(バージョンアップ時・不定期・1回ずつ)

**送信条件:** P-001に内容更新(v1.x)があった場合のみ。バージョンごとに1回・催促なし。実際の送信はマメが担当。

**件名:** 「PixelYen 完全ガイド」をアップデートしました(v<バージョン番号>)

```
<お名前>様

PixelYenのムスビです。
「PixelYen 完全ガイド」をアップデートしました(v<バージョン番号>)。

以前ご購入いただいた方への更新は追加費用なしでご提供しています。
以下から最新版をダウンロードしてください:
https://pixelyen.github.io/ai-company/download/guide-v1/

主な変更点: <変更内容>

引き続きよろしくお願いします。

ムスビ(PixelYen カスタマーサクセス)
```

**禁止事項:** 「大幅改善!」等の誇大表現。変更内容は事実のみ記載。

---

## 送信ルールまとめ

| タイミング | 内容 | 回数上限 |
|---|---|---|
| 購入後24h以内 | お礼 + ダウンロードURL | 1回 |
| 購入後3日 | 活用確認 | 1回 |
| 購入後7日 | レビュー依頼 | 1回(催促なし) |
| バージョンアップ時 | アップデート案内 | バージョンごとに1回 |

- 返金申請が来た場合: 通2以降はスキップし、マメに引き継ぐ
- 顧客から返信が来た場合: 内容に応じてムスビが個別対応(値引き・定型外対応は禁止)
- 顧客情報はフォロー目的以外に使わない(第3条🔴)

---

## マメとの連携

- **実際の送信**: 通1〜通4はすべてマメが手動送信。ムスビはテンプレと判断基準を管理。
- 返金・問い合わせはマメが担当(skills/refund-reply/SKILL.md)
- 購入者の声(感謝・改善要望)はムスビが記録し、経営会議報告材料に加工する
- 顧客フォローのステータスは `state/cs_log.md` で管理(スケルトン作成済み)
