# 第5章 プロンプト集 — 「社員」を作る6つの要素と、コピーして使える雛形

## この章でやること

ここまでの章で、会社の骨組み(リポジトリ・憲法・自走の仕組み・つまずき対処)は一通り揃いました。残るいちばん大事な部品が、**社員そのもの**です。この会社の「社員」は、人ではなく**ロールプロンプト**——「あなたはこういう役割で、こう判断し、こう報告する人です」と書いた一枚の指示書——でできています。

社員の出来は、この指示書の出来でほぼ決まります。だから私たちは、社内の品質基準で**「あなたはCEOです」だけの一行プロンプトを禁止**しています。それだけでは、AIはその場その場で都合よく振る舞い、判断がぶれ、報告の形もバラバラになるからです。

この章では、私たちが全社員に必須としている**6つの要素**を一つずつ説明し、最後に**そのままコピーして使える雛形**を置きます。読んで理屈を知るだけでなく、雛形を埋めれば今日あなたの会社の一人目を作れる、という形にしました。

## なぜ「6要素」なのか — 一行プロンプトが失敗する理由

一行の指示(「あなたは経理担当です。よろしく」)で動かすと、次のことが起きます。

- **判断がぶれる**——似た状況で、あるときは慎重に、あるときは大胆に振る舞う。基準が書いていないので、その場の勢いで決めてしまう
- **報告がバラバラ**——毎回ちがう形式で報告してくるので、後から比較・集計できない
- **越権する**——決めていいことと、人間に諮るべきことの線引きが無いので、勝手に重い判断をしてしまう
- **終わりが曖昧**——「どこまでやれば完了か」が無いので、やり残しや、逆に無駄な動きが出る

6つの要素は、この4つの失敗をふさぐために選んだものです。**役割と権限**で越権を防ぎ、**判断基準**でぶれを止め、**出力フォーマット**で報告を揃え、**良い例・悪い例**で振る舞いの当たりを付け、**禁止事項**で踏んではいけない線を引き、**完了条件**で終わりを定義する——順番にも意味があります。

## 6つの必須要素

### 1. 役割と権限(何を決めてよいか)

その社員が**何者で、どこまで自分で決めてよいか**を書きます。ここがいちばん大事です。「決めてよいこと」と「決めてはいけないこと(人間や上位の会議に諮ること)」を**両方**明記してください。片方だけだと、必ずどちらかに寄って事故ります。

- 良い書き方:「価格の**承認**はしてよい。ただし憲法・組織の**変更**はしてはいけない(提案のみ)」
- 悪い書き方:「経営のことはだいたい任せる」(範囲が無限で、越権の温床になる)

### 2. 判断基準(迷ったときの優先順位)

**何と何がぶつかったとき、どちらを採るか**を、順位で書きます。順位にするのがコツです。「品質も速度も大事」では、ぶつかったときに決められません。

- 私たちのCEOの判断基準は一行です:**「利益 > 速度 > 見栄え」**。迷ったら利益、それでも迷ったらより慎重な側を採り、記録に残す
- この一行があるおかげで、CEOは「動き続けること」と「利益を出すこと」がぶつかったとき、後者を選べます

### 3. 出力フォーマット(報告・成果物の型)

**報告や成果物を、どんな形で出すか**を、テンプレートとして固定します。ここを決めておくと、毎回の報告が同じ形で揃い、後から並べて比較できます。数字を必ず入れる欄を作っておくと、都合の悪い数字も自動的に表に出ます(隠せない設計)。

### 4. 良い例・悪い例(各1つ以上)

**具体的なシーンで「こう振る舞ってほしい/ほしくない」**を、1つずつ以上見せます。抽象的な指示より、実例1組のほうが、AIははるかに正確に狙いを掴みます。悪い例には「なぜダメか」を短く添えてください。

- 良い例:「候補Bのみ採用。理由=検索需要が実測でき、制作コストが2セッション以内。AとCは需要根拠が推測のため見送り」
- 悪い例:「全部やりましょう!勢いが大事!」(×=数字の根拠なし、コスト概念なし、記録なし)

### 5. 禁止事項(該当ルールを参照)

**絶対に踏んではいけない線**を、明示します。会社の憲法(基本ルール)がある場合は、**どの条文に基づく禁止か**まで書くと、AIが自分で理由を辿れて、判断の一貫性が上がります。私たちの場合は「現金支出は禁止」「収益の保証・示唆は禁止」「都合の悪い数字の隠蔽は禁止」などを、条文番号付きで各社員に書いています。

### 6. 完了条件(何をもって終わりとするか)

**そのセッションを「終わってよい」と判断する基準**を、チェックリストで書きます。「◯◯を確認した/△△を更新した/□□へ報告した」を全部満たしたら完了、という形です。これが無いと、やり残しにも、やりすぎにも気づけません。特に無人運転では、**「終わる前に成果物の現物を確認する」を完了条件に必ず入れてください**(第4章で書いた「やったのに成果物が無い」を防ぐ最後の砦です)。

## コピーして使える雛形

以下をそのまま複製し、`< >` の中をあなたの会社の言葉で埋めれば、社員一人分のロールプロンプトになります。ファイルは、社員ごとに1枚ずつ用意するのがおすすめです。

```markdown
# <社員名> — <役職(何をする人か一言で)>

性格: <一言で。口癖があると判断の軸が伝わりやすい。例: 「それは利益に繋がるか?」>

## 1. 役割と権限(何を決めてよいか)
- この社員がすること: <担当業務を具体的に>
- 決めてよいこと: <自己判断でよい範囲を列挙>
- 決めてはいけないこと: <人間・上位会議に諮るべきこと。理由も>

## 2. 判断基準(迷ったときの優先順位)
- 優先順位: <A > B > C の形で。ぶつかったとき何を採るか>
- 迷ったら: <より慎重な側を採り、記録に残す 等の原則>

## 3. 出力フォーマット(報告・成果物の型)
<毎回この形で報告する、というテンプレを固定。数字を入れる欄を必ず作る>
・<項目1>: <…>
・<項目2>: <…>
・数字: <売上/コスト/進捗など、隠せない実数を入れる欄>
・次回: <次の自分への申し送り>

## 4. 良い例・悪い例
- 良い例: <具体シーンでの理想の振る舞い(理由つき)>
- 悪い例: <やってほしくない振る舞い + なぜダメか>

## 5. 禁止事項
- <禁止1(できれば根拠ルール/条文を併記)>
- <禁止2>
- <禁止3>

## 6. 完了条件(すべて満たしたら終了)
- [ ] <確認すべきことを処理した>
- [ ] <更新すべき記録を更新した>
- [ ] <終わる前に成果物の現物を自分で確認した>
- [ ] <所定の場所へ所定の形式で報告した>
```

## 実例 — 私たちのCEOのロールプロンプト(抜粋)

雛形が実際どう埋まるか、私たち自身のCEO(社内では「経営の最終決定者」)の指示書を、要点だけ抜き出して見せます。値は各要素の中身のイメージです。

- **役割と権限**:「憲法の範囲内で社内の最終決定権を持つ。決めてよい=タスクの優先順位・商品化のGO/STOP・価格の承認。決めてはいけない=憲法や組織の**変更**(提案のみ)・社員の任免の確定・現金支出(物理的に不可能)」
- **判断基準**:「**利益 > 速度 > 見栄え。** それは利益に繋がるか? 繋がらないなら、やらない勇気を持つ。可逆な打ち手は速く、不可逆な打ち手は一晩寝かせる」
- **出力フォーマット**:「日次報告は『判断/割振/数字(今月売上・純利益)/次回』の4行。**数字の欄があるので、売上ゼロならゼロと書く**しかない」
- **良い例・悪い例**:上の「4」で挙げた候補選定の例が、そのまま入っています
- **禁止事項**:「現金支出・借入・外部登録(禁止)/収益の保証・示唆・誇大表現(禁止)/都合の悪い数字の隠蔽(禁止)」——それぞれ憲法の条文番号つき
- **完了条件**:「人間からの指示を処理した/承認結果を反映した/各社員へ割り振った/数字を確認した/決定を記録した/所定チャンネルへ報告した——を全部満たしたら完了」

この6要素が埋まっていれば、CEOは毎回同じ軸で判断し、同じ形で報告し、越権せず、やり残さない——**別々のセッションで動いても、同じ「人格」として振る舞える**ようになります。社員が増えても会社が破綻しないのは、全員がこの型で書かれているからです。

## つまずきやすい点(正直な補足)

> 私たち自身の失敗から言うと、**いちばん軽視されがちなのが「6. 完了条件」**です。役割や判断基準は気合いを入れて書くのに、「どこで終わるか」を書き忘れる。すると、AIは「作業した」と報告して満足し、成果物を保存し忘れたり、記録の更新を飛ばしたりします。第4章で書いた「やったのに成果物が無い」問題は、完了条件に**「終わる前に現物を自分で確認する」**の一行が無かったことも一因でした。6つのうち、完了条件だけは絶対に省かないでください。

> もう一つ。**6要素はコピペで似せるより、その会社の言葉で埋めるほど効きます。**判断基準の「利益 > 速度 > 見栄え」は私たちの会社の価値観であって、あなたの会社では別の順位かもしれません。雛形は骨、中身はあなたの会社の魂です。

## この章のまとめ

- 社員=ロールプロンプト。一行プロンプトは、判断のぶれ・報告のばらつき・越権・曖昧な終わりを生む
- 必須は6要素:**①役割と権限 ②判断基準 ③出力フォーマット ④良い例・悪い例 ⑤禁止事項 ⑥完了条件**
- 役割と権限は「してよい/してはいけない」を**両方**書く。判断基準は**順位**で書く。禁止事項は**根拠ルール**を添える
- 完了条件には必ず**「終わる前に成果物の現物を確認する」**を入れる(無人運転の最後の砦)
- 雛形は骨、中身はあなたの会社の言葉。コピーして埋めれば、今日一人目を作れる
